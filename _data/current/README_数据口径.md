# 共享数据层口径说明

本目录是《运营成本穿透看板1.0》的当前共享数据层，来源为 `_versions\V11`。V11 已由 `_versions\V10.4` 复制生成，因此本目录数据与 V10.4 封存成本口径保持一致。

## 本轮处理方式

本轮采用安全方案 A：

- 已创建 `_data\current`。
- 已从 `_versions\V11` 的运行时数据对象抽取共享数据层。
- `_versions\V11` 电脑端暂时保持原样，不强行改为读取 `_data\current`。
- 后续可在 V11.1 中做电脑端共享数据读取改造，V20.0 手机端可直接以 `_data\current` 为数据起点。

选择安全方案的原因：V10.4/V11 当前页面、数据生成、V9.2 兼容层和审计函数仍集中在 `app.js` 中，一次性拆分加载路径会提高页面不可用风险。本轮目标是工程整理和数据层搭设，不冒险重构电脑端展示链路。

## 文件说明

- `data-core.js`：核心数据表，包括项目、场馆、活动、合同、合同清单、工程量、进度结算、财务一体化、预算、固定成本池、分摊明细、`FACT_COST_DETAIL` 等。
- `data-views.js`：由核心数据派生的页面视图，包括首页项目汇总、月度成本汇总、项目科目树、活动成本、活动按科目成本、横向对比、成本明细追溯。
- `data-dictionary.js`：预算基准、月份权重、字段清单、事实表口径和数据字典。
- `data-audit.js`：关键金额、行数、抽取校验结果和本轮是否改造读取路径的决策说明。

## 核心口径

- 首页、月度页、项目页默认成本金额只统计 `fact_type = raw_cost`。
- 活动页和活动详情页用于解释活动经营消耗，可统计 `raw_cost + allocated_cost`。
- `allocated_cost` 只进入活动成本解释，不重复计入首页、月度页、项目页总成本。
- `FACT_COST_DETAIL` 行数保持 9323 行。
- 公司全年预算保持 24098.86 万元。
- 成本数据、预算数据、`raw_cost` / `allocated_cost` 口径均未重编、未改写。
